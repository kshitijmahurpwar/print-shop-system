# PROJECT RULES

## Architecture

* Separate frontend and backend
* REST API architecture
* Mobile-first design
* Local network based

## Customer Features

* Upload file
* Choose print options
* Generate token
* Track status

## Vendor Features

* Pending queue
* File preview
* Status management

## Statuses

* Pending
* Approved
* Printing
* Completed
* Rejected

## Restrictions

DO NOT build:

* Login system
* Payments
* Notifications
* AI features
* Multi-vendor support

## Design Rules

* Simple UI
* Beginner-friendly code
* Modular structure
* Responsive layout
